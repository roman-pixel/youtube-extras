# Шорткаты плеера

## Что делает

Добавляет горячие клавиши для кнопок панели быстрых действий в полноэкранном режиме YouTube. Каждый шорткат отображается как бейдж в нативном тултипе соответствующей кнопки.

## Шорткаты

| Клавиши | Действие |
|---------|----------|
| `Shift+C` | Открыть / скрыть комментарии |
| `Shift+L` | Лайк / убрать лайк |
| `Shift+D` | Дизлайк / убрать дизлайк |
| `Shift+S` | Поделиться |

Все шорткаты работают на любой раскладке клавиатуры — используется `e.code` (физическая позиция клавиши), а не `e.key` (символ).

## Как работает

**Определение полноэкранного режима**
YouTube добавляет атрибут `is-fullscreen` на элемент панели комментариев при входе в fullscreen. Это надёжнее, чем `document.fullscreenElement`, который возвращает `null` когда YouTube использует собственную реализацию полноэкранного режима.

**Поиск кнопок**
Все кнопки находятся внутри `.ytp-fullscreen-quick-actions`. Лайк и дизлайк ищутся по кастомным элементам-обёрткам (`like-button-view-model`, `dislike-button-view-model`) — они языконезависимы. Комментарии ищутся по `aria-label` с fallback на уникальный SVG path иконки. Поделиться — по `aria-label`.

**Триггер действий**
Каждый шорткат кликает нативную кнопку YouTube, не манипулируя DOM-состоянием напрямую. Это сохраняет консистентность внутреннего состояния YouTube (загрузка контента, анимации, счётчики).

**Бейджи в тултипах**
YouTube использует один общий элемент `yt-popover` для тултипов всех кнопок. `MutationObserver` следит за ним. Отслеживание hover через `mouseover` определяет, над какой кнопкой находится курсор — поэтому бейджи вставляются независимо от языка и состояния кнопки (например, «Нравится» vs «Больше не нравится»).

## Файлы

- `modules/playerShortcuts.js` — поиск кнопок и регистрация шорткатов
- `modules/shortcutCore.js` — общие утилиты, используемые этим модулем
