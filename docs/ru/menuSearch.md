# Поиск в меню

Строка поиска, инжектируемая в меню настроек плеера (`.ytp-settings-menu`) для любой панели длиннее `SEARCH_MIN_ITEMS` (5). Покрывает субтитры, авто-перевод и любой другой длинный список, который YouTube может показать в этом меню.

## Поведение

- **Инжект.** Между заголовком панели и списком вставляется обёртка: поле ввода + счётчик совпадений + кнопки prev/next.
- **Фильтр.** Подстроковый матч, регистронезависимо. Совпадения подсвечиваются через `<mark>`. У текущего совпадения отдельный цвет, к нему скроллит `scrollIntoView({ block: "nearest" })`.
- **Навигация.** `Enter` → следующее, `Shift+Enter` → предыдущее. Кнопки `‹` / `›` делают то же самое. Кнопки неактивны, когда совпадений нет.
- **Фокус.** `Shift+F` ставит фокус в поле активной панели. `Escape` снимает фокус. Распространение клавиатурных событий внутри input останавливается, чтобы не срабатывали хоткеи YouTube/браузера.
- **Sticky.** Строка поиска использует `position: sticky` и остаётся видимой при прокрутке списка.
- **i18n.** Тексты placeholder и тултипов берутся из `t()` в `i18n.js`. Русский для браузерных локалей `ru-*`, английский для остальных.

## API

```js
const controller = attachSearch({
  container: ".ytp-settings-menu",
  itemSelector: ".ytp-menuitem",
  labelSelector: ".ytp-menuitem-label",
  placeholder: "Поиск",
});
```

`attachSearch` ждёт появления контейнера, следит за ним через `MutationObserver` и обрабатывает каждую подходящую панель. Возвращает `{ focusActive() }`.

`focusActive()` фокусирует input видимой панели. Если input ещё не создан, запрос ставится в очередь и срабатывает, как только панель смонтируется. Используется `Shift+S` для автофокуса сразу после открытия меню (`focusPlayerMenuSearch()` в `playerMenuSearch.js`).

## Файлы

- `modules/search.js` — универсальный инжектор
- `modules/playerMenuSearch.js` — подключение к меню настроек плеера, `focusPlayerMenuSearch()`
- `styles/search.css` — стили строки поиска
- `styles/tooltip.css` — кастомный тултип на кнопках навигации
